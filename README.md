# 학습 지식 그래프 (knowledge-graph)

> 🔗 **라이브**: https://knowledge-graph-lyart.vercel.app

> 흩어진 학습 자료(`.md`·velog 글)에서 개념을 자동으로 뽑아 **문서·개념 이분 그래프로 자동 연결**, *따로 배운 자료들이 알고 보니 같은 개념으로 이어져 있다는 걸* 눈으로 발견하게 하는 도구. Obsidian과 달리 `[[ ]]`를 안 써도 연결이 뜬다.

핵심 가치는 **관계 발견**이다. 여러 자료를 넣으면 손 하나 안 대고 개념이 추출돼 그래프가 떠, "내 React 글과 JS 글이 *클로저*로 이어져 있었네" 같은 *내가 안 그린 연결*을 발견한다.

---

## 무엇을 / 왜

- **무엇**: 여러 `.md`·velog 글을 넣으면 → 개념을 자동 추출·정규화 → **문서 노드 + 개념 노드 + 문서→개념 멤버십 엣지**를 자동으로 그리는 웹앱. 두 문서가 같은 개념을 공유하면 그 개념을 다리 삼아 저절로 이어진다.
- **왜**: 학습 기록(주간 로그·velog 글)이 흩어져 있어 *무엇을 배웠고 어떻게 이어지는지* 한눈에 보기 어렵다. 그 흩어진 기록을 하나의 지도로 모은다.
- 취업 포트폴리오 + 도구/워크플로우 학습 + 본인이 쓸 실용 도구 + 멘토 과제를 겸한다.

> 2026-06-12 **이분 그래프로 피벗**([ADR 0013](docs/decisions/0013-bipartite-model-pivot.md)): 옛 버전은 개념만 점으로 두고 *손으로* 선을 그었으나, 그건 "내가 이미 아는 연결"만 그려져 발견이 안 일어났다. 이제 문서도 점이고, 연결은 내용에서 **자동**으로 나온다.

---

## C-MVP — "드롭하면 떠오르는 발견 지도" (이분 그래프)

여러 자료를 넣으면 자동으로 그래프가 떠, *내가 안 그린 연결*을 발견한다. 손으로 긋는 단계는 없다.

| 기능 | 역할 |
| --- | --- |
| 다중 입력 누적 | 여러 `.md` 올리기 + velog 주소 여러 개 → 각 자료가 **문서 노드** |
| 개념 자동 추출 | 글에서 키워드(제목·인라인 코드·볼드) 자동 추출 (사람 채택 단계 없음) |
| 표기 정규화 | 소문자·공백 통일 → *같은 표기 = 같은 개념 노드*로 자동 병합 |
| 이분 그래프 자동 연결 | **문서→개념 멤버십 엣지** 자동 생성 → 공유 개념이 자료들을 이음 |
| 노드 종류 구분 | 문서 vs 개념을 모양·색으로 구분 |
| 움직이는 그래프 | 끌기·확대축소·이동 |
| 검색 + 거르기 | 노드가 많아지면 검색·필터 |
| 고립 강조 | 아무와도 안 엮인 자료·개념을 강조 → "아직 지식망에 안 엮임" 힌트 |
| 저장 | 로그인 → 서버(Supabase) 저장·복원 (멀티기기) |

## 진행 상황 & 로드맵

> 우선순위·"안 만들기로 한 결정"의 **기준 문서는 [docs/roadmap.md](docs/roadmap.md)**. 아래는 요약.

**완료**: 옛 MVP → 배포(Vercel) → DB·인증(Supabase) → dev/prod DB 분리. 라이브 운영 중.

**진행 중**: **이분 그래프 피벗**(C-MVP) — 설계 완료(ADR 0013 · spec 2026-06-12), 구현 차례. *옛 로드맵의 "엣지 자동 추천"은 이 피벗으로 흡수됨.*

**다음 순서**:

1. **추출 정교화** — 줄글에서 핵심 개념만 더 잘 뽑기 (파이프라인 맨 앞이라 노이즈가 다 전파됨).
2. **UI 구축 (Claude 디자인 도구 — claude.ai/design)** — 거친 기능 UI를 제대로 된 비주얼(레이아웃·색·타이포·컴포넌트)로. 라이브 산출물의 첫인상 = 포폴 임팩트.
3. **능동 학습 루프 ★** — 퀴즈로 이해도 측정(개념 노드 클릭 → 질문 → 채점 → 꼬리질문, **LLM은 여기서 합류**) → 이해도 표시(크기·테두리) → 다음 학습 추천(고립·약연결 노드).

> 성장 스토리: 처음엔 되돌아보는 '발견 지도'로 만들고, 비주얼을 다듬은 뒤, 퀴즈를 붙여 이해도를 측정하는 '능동 학습 도구'로 키운다.

## 기술 스택

Next.js 16 · React 19 · TypeScript 5 · Tailwind 4 · zustand · react-force-graph-2d · Supabase(DB·인증)

- 순수 로직(추출·**정규화·그래프 빌드**·그래프 연산)은 `src/lib`에 두고 단위 테스트(TDD)로 검증.
- **테스트 3층**: Vitest(유닛·로직) + RTL(폼류 컴포넌트) + **Playwright**(E2E — 실제 브라우저로 canvas 렌더·흐름 게이트). canvas는 jsdom이 못 그리므로 "그래프가 진짜 뜨나"는 Playwright 몫.
- velog 글은 브라우저에서 직접 못 부르므로(CORS) 서버 라우트(`/api/velog`)가 대신 호출.
- 그래프 화면은 클라이언트 컴포넌트(canvas) — `dynamic import + ssr:false`.
- 게이트: ESLint/Prettier + Husky + commitlint + lint-staged.
