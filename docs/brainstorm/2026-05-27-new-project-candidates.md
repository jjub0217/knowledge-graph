# 새 프로젝트 후보 — 상품·게시판 외 도메인

> 2026-05-27 브레인스토밍 세션. 본인 자산·습관에서 도출한 후보 4개.

## 후보 1. 시간 사용 시각화 (Where Did My Time Go)

> "내 시간이 어디로 가는가" — 개발자·학습자용 시간 트래킹 + 시각화

```
[입력]                    [처리]               [출력]
타이머·태그 (수동)       ┐                 ┌── 일별·주별 분배 차트
GitHub commit 시각    │── 카테고리 분류 ────┤    (개발·학습·블로그·휴식)
캘린더 (선택)           │                 │
                     ┘                 └── 패턴 인사이트
                                            ("월요일은 학습이 적다" 등)
```

- 본인 강점: ★★★★ (시각화·측정 먼저)
- 포폴: ★★★★ (차트·UX + 시간 데이터 모델링)
- 흔함도: 중간 (RescueTime·Toggl 등 있지만 개발자 학습 흐름 특화 자리는 있음)
- 규모: 3~4주

## 후보 2. 의사결정 매트릭스 도구 ⭐ (Decision Studio)

> 본인이 매번 하는 트레이드오프 비교를 시각적 도구로 — 노트북·이직·도구·기술스택 결정용

```
[입력]                              [처리]               [출력]
옵션 N개 (예: 도구 A·B·C)      ┐                  ┌── 비교 매트릭스
기준 N개 (예: 가격·러닝커브)     │── 가중치 계산 ──────┤    (★ 점수)
각 셀에 점수·메모              │                  │
                           ┘                  ├── 시각화 (레이더·바)
                                              │
                                              └── "근거 노트"
                                                (결정 후 회고용)
```

- 본인 강점: ★★★★★ (트레이드오프·시각화 = 본인 패턴 그대로)
- 포폴: ★★★★ (UX 디테일·상태 관리·시각화 라이브러리)
- 흔함도: 낮음 (Notion·Excel로 하지만 전용 도구는 드뭄)
- 규모: 2~3주 (작게 시작 가능)
- 보너스: 본인이 본인 결정을 진짜로 이걸로 함 — 면접에서도 "내가 만든 도구로 결정했어요" 어필

## 후보 3. 색상 접근성 스튜디오 (A11y Color Studio)

> 본인이 5/25 머지한 a11y 색상 대비 작업 → 그걸 도구화

```
[입력]                  [처리]                    [출력]
디자인 토큰         ┐                         ┌── 대비 매트릭스
브랜드 컬러         │── WCAG AA·AAA 검사 ──────┤    (전경×배경 조합)
                 ┘                         │
                                           ├── 대체 제안
                                           │   ("이 조합은 4.5:1 부족,
                                           │    이렇게 바꾸면 OK")
                                           │
                                           └── 시뮬레이터
                                               (색맹 유형별 미리보기)
```

- 본인 강점: ★★★★ (최근 실전 경험 있음)
- 포폴: ★★★★★ (a11y는 프론트 면접 차별화 강력)
- 흔함도: 낮음 (단순 검사는 많지만 디자인 시스템 + 시뮬레이터 + 제안까지는 드뭄)
- 규모: 2~3주

## 후보 J. 학습 지식 그래프 (Learning Knowledge Graph)

> 학습한 개념들을 노드·엣지로 — Obsidian의 그래프뷰를 학습 흐름 특화로

```
[입력]                      [처리]               [출력]
학습 로그 키워드         ┐                  ┌── 인터랙티브 그래프
블로그 글              │── 키워드 추출 ──────┤    (D3·react-force-graph)
관계 (수동 or 자동)     │   관계 매핑         │
                    ┘                   ├── "이 개념 → 다음 추천"
                                        │
                                        └── 학습 깊이 시각화
                                            (이해도 색상 그라데이션)
```

- 본인 강점: ★★★★ (학습 자산 활용)
- 포폴: ★★★★ (그래프 시각화 = 어려운 영역)
- 흔함도: 낮음 (학습 그래프 도구 드뭄)
- 규모: 3~5주 (그래프 시각화가 난이도 있음)
