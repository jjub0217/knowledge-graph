# 0006. 그래프 그리는 라이브러리 = react-force-graph

- 상태: 채택
- 날짜: 2026-05-29

## 맥락 (왜 이 결정이 필요한가)
"그래프 화면" 모듈(설계 문서 §7)이 점·선을 실제로 화면에 그린다. 무엇으로 그릴지 정해야 한다. 요구: React/Next.js 연동, 포스(force) 레이아웃(덩어리가 자연스럽게 뭉치는 "지도" 미학 = 핵심 가치 '관계 발견'), 드래그·줌·팬, 손으로 엣지 그리기, 주제별 색·고립 점 강조, 수백 노드 규모, 입문자가 다룰 만함.

## 고려한 선택지
- A. **react-force-graph** — React 래퍼(내부에 `d3-force` 사용). 좋은 점: React에 거의 그대로 끼움, Obsidian풍 force 미학, 드래그·줌 기본, 색·크기 커스텀 쉬움. 아쉬운 점: 두 점을 이어 엣지를 만드는 편집은 직접 구현.
- B. **Cytoscape.js** — 그래프 편집·분석 특화, 엣지 그리기 확장·그래프 알고리즘 내장. 아쉬운 점: API가 무겁고 스타일이 자체 문법, 기본 미학은 덜 예쁨.
- C. **vis-network** — 노드·엣지 편집 UI 내장, 쉬움. 아쉬운 점: React 친화도 낮음(명령형), 스타일 유연성 낮음.
- D. **D3(d3-force) 직접** — 무한 커스텀·학습 가치 큼. 아쉬운 점: 드래그·줌·렌더를 다 직접 → MVP까지 가장 오래.

## 결정
**A. react-force-graph.**

## 근거 / 무엇을 얻고 무엇을 포기했나
- 핵심 가치가 "관계 발견(시각적 지도)"이고 사용자가 React를 학습 중이라, **미학 + React 통합 난이도**에서 가장 유리하다. 내부에 d3-force를 써서 Obsidian풍 클러스터가 그대로 나온다.
- 수동 엣지 그리기는 "노드 클릭 → 다른 노드 클릭 = 연결" 같은 작은 커스텀으로 충분하다.
- 얻은 것: 빠른 MVP + 원하는 미학 + 쉬운 React 통합. 포기한 것: Cytoscape/vis-network의 엣지 편집 기본 제공, D3 직접 구현이 주는 학습·포트폴리오 신호.

## 재검토 조건 (언제 바꾸나)
- 채용 타깃 공고 **여러 곳에서 D3·데이터 시각화가 반복**되면, "그래프 화면" 모듈을 **D3 직접 구현(d3-force + React)** 으로 전환을 검토한다. react-force-graph는 d3-force 래퍼라 D3 실력을 *증명*하진 못하므로, 그 경우 직접 작성이 포트폴리오 신호로 유리하다.
- "그래프 화면"이 인터페이스로 분리돼 있어(설계 문서 §7) 전환 시 다른 코드 영향이 작다.

## 결과 (이 결정으로 생기는 영향)
- "그래프 화면" 모듈은 react-force-graph로 구현한다. 의존성: `react-force-graph`.
- 점은 주제→색, 연결 수(degree)로 고립·약연결 강조. 엣지는 클릭 두 번으로 수동 연결(커스텀).
- 캔버스 기반이라 수백 노드 규모 성능은 무난(필요해지면 측정).
