# 0010. DB 저장·인증 도입 시점을 앞당김 (로드맵 7 → 2)

- 상태: 채택
- 날짜: 2026-06-05

## 맥락 (Context)
[ADR 0003](0003-framework.md)에서 MVP 저장은 **브라우저 안 저장(localStorage) + JSON 내보내기/가져오기**로 충분하다고 보고, 서버 DB는 "여러 기기·백업·공유가 필요해지면" 도입하기로 해 [roadmap.md](../roadmap.md)의 **맨 뒤(7번)**에 두었다.

MVP(Task 1~11)를 완주한 뒤, 개발자가 다음을 밝혔다: **"어차피 DB를 쓸 것이고, 그 가장 큰 이유는 포트폴리오 임팩트"**(백엔드·인증·DB를 다룰 줄 안다는 걸 면접·포폴에서 보여주기). 멀티기기 사용·데이터 안전도 부수 동기다. 이에 "localStorage로 충분하니 DB는 맨 뒤"라는 전제를 다시 보게 되었다.

## 고려한 선택지 (Options)
- **A. 현행 유지 (DB 7번 맨 뒤)** — 장점: 가볍고, 배포·추출·퀴즈 같은 제품 가치를 먼저. / 단점: 포폴의 풀스택 신호가 늦고, DB 전까지 멀티기기·데이터 안전이 공백.
- **B. DB를 배포보다도 먼저(1번)** — 장점: 처음부터 제대로 된 영속성. / 단점: 무거워 라이브 산출물이 늦어지고, "싸게 배포해 사용 데이터로 다음 타깃을 정한다"는 배포-먼저의 이점을 잃음.
- **C. 배포 1번 유지 + DB·인증을 2번으로** — 장점: 라이브 산출물을 즉시 확보하면서 포폴 핵심(풀스택)을 이른 단계에 두고, 멀티기기·데이터 안전도 해결. / 단점: localStorage 배포본을 곧 DB로 바꿔야 함(단, 호스팅·빌드 설정은 재사용되고 바뀌는 건 저장 계층뿐이라 손실이 작음).

## 결정 (Decision)
**C를 택한다.** 로드맵을 다음 순서로 재배치한다:

```
1. 배포  2. DB+인증  3. 추출 정교화  4. 퀴즈★  5. 이해도 표시  6. 다음 학습 추천  7. 연결선 자동 추천
```

- **DB 저장 + 인증**을 7번 → **2번**으로 앞당긴다.
- **배포는 1번 유지.** (싸고, 라이브 산출물을 즉시 만들며, DB가 와도 호스팅을 재사용한다.)
- **추출 정교화(3)를 퀴즈(4)보다 앞**에 둔다 — 퀴즈는 앱 성격을 바꾸는 가장 큰 피벗이라, 그 전에 입력(후보) 품질을 정비해 깨끗한 개념이 퀴즈로 흐르게 한다.

## 근거 / 트레이드오프 (Rationale)
- 주동기가 **포폴 임팩트**이므로, DB+인증은 단순 저장 편의가 아니라 **풀스택 역량을 보여주는 가장 선명한 산출물**로서 가치가 크다. 부수적으로 멀티기기·데이터 안전(캐시 삭제 시 유실 방지)도 함께 해결된다.
- 배포를 1번에 유지하는 이유: 배포 자체는 **싸고 즉시 포폴 한 칸을 채우며**, DB로 바꿀 때도 호스팅·빌드 설정은 그대로 쓰고 **교체 지점은 저장 계층(`storage.ts`) 하나**라 선행 비용이 거의 버려지지 않는다.
- DB는 **퀴즈(능동 학습)의 토대**가 된다 — 퀴즈가 매긴 '이해도'를 DB에 저장하면, 로드맵 퀴즈 항목의 "무거움(웹 직접)" 방식이 자연스러워진다. 그래서 DB가 퀴즈보다 앞에 오는 것이 순서상 맞다.
- 포기한 것: "가장 가벼운 길로 제품 가치(퀴즈 등)부터"라는 A의 단순함. 대신 풀스택 신호를 이른 단계에 확보.

## 결과 (Consequences)
- [roadmap.md](../roadmap.md)의 우선순위를 위 순서로 갱신한다.
- 이 결정은 [ADR 0003](0003-framework.md)의 **프레임워크·MVP 저장 선택(Next.js·localStorage)을 뒤집지 않는다** — localStorage는 MVP 저장으로 여전히 타당했다. 바뀌는 것은 **DB 도입 시점(우선순위)**뿐이다.
- **DB+인증은 큰 마일스톤**이다: 스키마·API·인증·보안·호스팅·비용을 별도로 설계해야 한다. 착수 시 새 ADR(스택·인증 방식 등)이 필요할 수 있다.
- localStorage → DB 전환의 **교체 지점은 `storage.ts`** (그 인터페이스를 유지하면 화면·스토어 변경을 최소화할 수 있다).
