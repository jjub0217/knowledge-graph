# 설계 문서 — 배포 + 예시 데이터 시드 (로드맵 1번)

- 날짜: 2026-06-07
- 상태: 초안 (사용자 검토 대기)
- 관련: 이슈 #33 · 로드맵 1번 · [ADR 0010](../decisions/0010-db-auth-priority.md)(다음은 2번 DB+인증)

> 쉬운 한국어로 작성. MVP를 라이브로 띄우는 "배포" 마일스톤의 청사진.

---

## 1. 한 줄 요약 / 목표

완성된 MVP를 **Vercel에 호스팅**해 면접관·블로그 독자가 URL로 바로 써보게 한다. 첫 방문자가 빈 화면 대신 **미리 채워진 예시 지식 그래프**를 보도록 시드(seed) 데이터를 넣는다.

## 2. 왜 (동기)

- 로드맵 1번. 싸게 끝나고 포폴 임팩트가 크며, 실제 사용 데이터가 다음 작업(추출 정교화)의 타깃을 정해준다.
- 앱은 브라우저 저장(localStorage)이라 **첫 방문자는 저장된 게 없어 빈 화면** → 도구의 가치(점·선·색·고립 강조)가 한눈에 안 보인다. 예시 데이터로 첫인상을 해결한다.

## 3. 범위 (이번에 하는 것 / 안 하는 것)

**한다**

- 예시 그래프 시드 (첫 방문 시 자동 로드)
- "비우기" 버튼 (예시를 지우고 직접 해보기)
- Vercel + GitHub 자동 배포 연결

**안 한다 (다음/별도)**

- DB·인증(로드맵 2번, ADR 0010) — 저장은 여전히 localStorage.
- 커스텀 도메인 — 기본 `*.vercel.app` 주소로 충분.
- 예시 "다시 불러오기" 버튼 — 비우기만. (필요해지면 후속)

## 4. 설계

### 4.1 예시 데이터 시드

- 예시 그래프를 **TypeScript 상수 파일** `src/lib/example-graph.ts`에 둔다(`Graph` 타입). fetch 없이 번들에 포함 — 가장 가볍고 타입 안전.
- 콘텐츠: 주제 2~3개(예: React·테스트·CSS) + **연결된 덩어리 + 고립 점 1개**를 넣어 색·연결·고립 강조가 한눈에 보이게 한다.

### 4.2 첫 방문 판별 (핵심 트릭)

- **"첫 방문" = localStorage에 저장 키가 아예 없음** / **"비우기 한 상태" = 키는 있고 내용이 빈 그래프**. 키 존재 여부로 둘을 구분한다.
- `storage.ts`에 헬퍼 `hasStoredGraph(): boolean`(키 존재 여부) 추가.
- 첫 로드 분기: `hasStoredGraph()`가 **거짓이면 예시 그래프**를, 참이면 저장본을 로드.
- 효과: **비우기 후에는 예시가 다시 안 튀어나온다**(빈 그래프를 저장 → 키 존재 → 재시드 안 함). 사용자 의도 존중.

### 4.3 "비우기" 버튼

- `Controls.tsx`에 추가. `confirm()`으로 한 번 확인한 뒤 그래프를 빈 상태로(`setAll([], [])`) → 저장도 빈 상태.

### 4.4 배포 (Vercel + GitHub)

- Vercel 대시보드에서 GitHub 저장소를 연결한다(**사용자가 직접** — 로그인 필요, 인터랙티브).
- Next.js 자동 감지 → 빌드 → 배포. **비밀값 없음**(환경변수 불필요).
- `/api/velog` 서버 라우트는 Vercel에서 **serverless 함수**로 동작 — 배포 후 실제 호출로 확인.
- main 브랜치 머지 시 자동 배포 + PR마다 미리보기 URL 생성.

## 5. 모듈 / 파일 변경

| 파일 | 변경 | 책임 |
| --- | --- | --- |
| `src/lib/example-graph.ts` | 신규 | 예시 그래프 상수(`Graph`) |
| `src/lib/storage.ts` | `hasStoredGraph()` 추가 | 첫 방문 판별(키 존재 여부) |
| `src/lib/storage.test.ts` | 테스트 추가 | `hasStoredGraph` 동작(TDD) |
| `src/app/page.tsx` | 첫 로드 분기 수정 | 첫 방문이면 예시, 아니면 저장본 |
| `src/components/Controls.tsx` | "비우기" 버튼 추가 | 그래프 비우기 |

## 6. 데이터 흐름 (첫 로드 분기)

```
페이지 첫 로드
   │
   ├─ hasStoredGraph() 거짓(첫 방문) → 예시 그래프 로드 → 화면에 표시
   └─ hasStoredGraph() 참            → 저장본 로드(빈 그래프 포함)
변경 시 → 항상 localStorage에 저장(키 생성됨)
비우기 → 빈 그래프 저장(키 존재 유지) → 다음 방문엔 빈 화면(재시드 안 함)
```

## 7. 에러 처리

- 예시 데이터는 우리가 만든 내부 상수라 검증 불필요(바깥 입력 아님).
- 기존 입구 검사(올린 .md·velog 응답·붙여넣은 주소·가져온 JSON)는 그대로.

## 8. 테스트 전략

- **순수 로직 = TDD**: `hasStoredGraph()`(키 있음/없음), 첫 로드 분기에서 "키 없으면 예시" 조건. 러너 = Vitest.
- **수동 검수**: 첫 방문 시 예시가 보이나, 비우기 동작, 배포된 사이트에서 `/api/velog`·골든패스.
- 예시 데이터 자체와 배포 인프라는 자동 테스트 대상 아님.

## 9. 열린 질문 / 미루는 것

- 예시 "다시 불러오기" 버튼: 이번엔 제외(비우기만). 필요해지면 후속.
- 커스텀 도메인·애널리틱스: 후속.

## 10. 용어 풀이

- **시드(seed) 데이터**: 처음에 미리 넣어두는 예시 데이터(씨앗).
- **serverless 함수**: 서버를 직접 안 띄우고, 요청이 올 때만 잠깐 실행되는 서버 코드 조각(Vercel이 `/api/velog`를 이렇게 돌린다).
- **미리보기(preview) 배포**: PR마다 자동으로 만들어지는 임시 URL — 머지 전에 그 변경을 실제로 눌러볼 수 있다.
